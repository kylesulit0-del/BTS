<objective>
Add a transparent touch-intercepting overlay to video cards that captures all touch events from the iframe, detects taps to toggle play/pause with visual feedback, and allows swipe gestures to bubble up to parent handlers.

Purpose: Cross-origin iframes (YouTube, TikTok) swallow all touch events, preventing vertical/horizontal swipe navigation. The overlay solves this while also enabling tap-to-play/pause with a TikTok-style feedback icon.

Output: Video cards that respond to swipe and tap gestures identically to image/text cards, with play/pause icon feedback on tap.
</objective>

**SnapCardVideo.tsx — Active state (the return block with the iframe):**

1. Remove `onClick={togglePlayPause}` from the outer `.snap-card-video` div. Tap detection moves to the overlay.

2. Add a transparent overlay div AFTER the iframe element and BEFORE the mute button. This div intercepts all touch/pointer events that would otherwise be swallowed by the cross-origin iframe:
   ```tsx
   <div
     className="snap-video-touch-overlay"
     onTouchStart={handleOverlayTouchStart}
     onTouchEnd={handleOverlayTouchEnd}
   />
   ```

3. Implement tap detection using touch events (NOT onClick — onClick has 300ms delay on mobile and can't distinguish taps from swipes). Use refs to track touch start position and time:
   - `TAP_MAX_DISTANCE = 10` (px) — if finger moves more than this, it's not a tap
   - `TAP_MAX_DURATION = 300` (ms) — if touch lasts longer than this, it's not a tap
   - On `touchstart`: record `clientX`, `clientY`, `Date.now()` in a ref
   - On `touchend`: compute dx, dy, dt. If `dx < TAP_MAX_DISTANCE && dy < TAP_MAX_DISTANCE && dt < TAP_MAX_DURATION`, call `handleTap()`
   - **CRITICAL:** Do NOT call `e.stopPropagation()` or `e.preventDefault()` in either handler. Touch events must bubble up to the parent containers where `useVerticalPaging` (on `.snap-feed` container via addEventListener) and `useSwipeGesture` (on `.snap-card-layout` via React onTouchStart/Move/End) listen.

4. Add play/pause feedback icon that appears on tap and fades out (TikTok/YouTube style):
   - Add state: `const [tapIcon, setTapIcon] = useState<'play' | 'pause' | null>(null)` and a timeout ref
   - `handleTap()` calls `togglePlayPause()`, then sets `tapIcon` to the NEW state (if was playing, show 'pause' icon; if was paused, show 'play' icon — use `isPlaying` from useSnapVideo to determine: `setTapIcon(isPlaying ? 'pause' : 'play')`)
   - Clear any existing timeout, set a new one to clear tapIcon after 800ms
   - Render the icon when tapIcon is not null:
     ```tsx
     {tapIcon && (
       <div className="snap-video-tap-icon" key={tapIcon}>
         <svg width="64" height="64" viewBox="0 0 64 64" fill="none">
           <circle cx="32" cy="32" r="32" fill="rgba(0,0,0,0.5)" />
           {tapIcon === 'play' ? (
             <polygon points="26,20 26,44 46,32" fill="white" />
           ) : (
             <g fill="white">
               <rect x="22" y="20" width="8" height="24" rx="2" />
               <rect x="34" y="20" width="8" height="24" rx="2" />
             </g>
           )}
         </svg>
       </div>
     )}
     ```
   - The `key={tapIcon}` ensures the animation restarts when tapping rapidly between play/pause.
   - Place this element AFTER the overlay div (higher in render order) with `pointer-events: none` in CSS so it doesn't intercept touches.

5. Change YouTube iframe URL from `controls=1` to `controls=0` in `buildIframeSrc()`. Per CONTEXT.md: "YouTube's built-in controls are hidden, all interaction goes through our gesture handler." The custom progress bar, mute button, and play/pause icon replace native controls. Users cannot interact with YouTube's controls through the overlay anyway.
