  <action>
1. In `packages/server/src/db/schema.ts`, add to the `contentItems` table:
   - `moderationStatus: text('moderation_status').notNull().default('raw')` — values: 'raw' | 'pending' | 'approved' | 'rejected'
   - `moderatedAt: integer('moderated_at', { mode: 'timestamp' })` — when the LLM last processed this item
   - Add `index('idx_moderation_status').on(table.moderationStatus)` to the indexes array

2. In `packages/server/src/db/schema.ts`, add a new `pipelineRuns` table:
   ```
   id: integer PK autoIncrement
   startedAt: integer timestamp, notNull
   completedAt: integer timestamp, nullable
   itemsProcessed: integer, default 0
   itemsApproved: integer, default 0
   itemsRejected: integer, default 0
   inputTokens: integer, default 0
   outputTokens: integer, default 0
   estimatedCost: text (USD string like "0.000042")
   provider: text, notNull (e.g., 'openai/gpt-4.1-nano')
   status: text, notNull, default 'running' — values: 'running' | 'success' | 'fallback' | 'error'
   error: text, nullable
   fallbackMode: integer boolean, default false
   ```

3. In `packages/server/src/db/schema.ts`, add a new `pipelineDecisions` table:
   ```
   id: integer PK autoIncrement
   contentItemId: integer, notNull
   runId: integer, notNull
   relevant: integer boolean, notNull
   safe: integer boolean, notNull
   contentType: text, nullable
   decision: text, notNull — values: 'approved' | 'rejected'
   decidedAt: integer timestamp, notNull
   ```

4. Create the SQL migration file `packages/server/drizzle/0002_phase7_moderation_pipeline.sql`:
   - ALTER TABLE content_items ADD COLUMN moderation_status TEXT NOT NULL DEFAULT 'raw'
   - ALTER TABLE content_items ADD COLUMN moderated_at INTEGER
   - CREATE INDEX idx_moderation_status ON content_items(moderation_status)
   - **CRITICAL:** UPDATE content_items SET moderation_status = 'approved' WHERE moderation_status = 'raw' — backfill existing items so the feed doesn't go empty when Plan 02 adds the approved-only filter
   - CREATE TABLE pipeline_runs (all columns as above)
   - CREATE TABLE pipeline_decisions (all columns as above)

5. Update `packages/server/drizzle/meta/_journal.json` to add the new migration entry (version 2, tag matching drizzle naming convention — look at existing entries for the pattern).

6. In `packages/shared/src/types/feed.ts`:
   - Add `moderationStatus?: string` to the `FeedItem` interface (optional since the feed API currently doesn't return it, and frontend doesn't need it for basic display — but admin views will use it)
   - Ensure `ContentType` still includes the 7 types matching the CONTEXT.md decision: 'news' | 'fan_art' | 'meme' | 'video' | 'discussion' | 'translation' | 'official' (already correct)

Note: The `contentType` column already exists on `content_items` from Phase 5 schema. The migration does NOT need to add it. Phase 7 simply populates it via the LLM pipeline.
