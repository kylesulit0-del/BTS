    Build the API routes and wire everything into the server entry point.

    **packages/server/src/routes/feed.ts:**
    Export an async function `registerFeedRoutes(server: FastifyInstance)` that registers:

    `GET /feed` -- Paginated feed endpoint:
    - Query params: `cursor` (optional string), `limit` (optional string, default "50"), `source` (optional string filter, e.g., "reddit")
    - Implementation using Drizzle:
      1. Parse limit as integer, clamp to 1-100 range, default 50 (per user decision)
      2. Build conditions array:
         - If cursor provided: filter where id < parseInt(cursor) (cursor is the last item's ID -- for descending order pagination)
         - If source provided: filter where source = source
      3. Query: `db.select().from(contentItems).where(and(...conditions)).orderBy(desc(contentItems.scrapedAt)).limit(limit + 1)`
      4. If results.length > limit: hasMore = true, slice to limit items
      5. nextCursor = last item's id (as string) if hasMore
      6. Count total items (with source filter if applicable): `db.select({ count: sql\`count(*)\` }).from(contentItems).where(...)`
      7. Map DB rows to FeedItem response shape (convert timestamp integers to ISO 8601 strings for publishedAt, scrapedAt)
      8. Return: `{ items, nextCursor, hasMore, total }`
    - Response matches the FeedResponse type from @bts/shared

    **packages/server/src/routes/scrape.ts:**
    Export `registerScrapeRoutes(server: FastifyInstance)`:

    `POST /scrape` -- Manual scrape trigger (per user decision):
    - Calls `runAllScrapers(db, scrapers)` with the Reddit scraper
    - Returns `{ status: 'ok', results: { itemsFound, itemsNew, itemsUpdated } }`
    - If scrape is already running, return `{ status: 'already_running' }` (simple flag-based guard)

    **packages/server/src/routes/health.ts:**
    Export `registerHealthRoutes(server: FastifyInstance)`:

    `GET /health` -- Source health endpoint:
    - Query the last scrape_run for each unique source
    - Return `{ sources: [{ source, lastRun, status, itemsFound }], uptime: process.uptime() }`

    **packages/server/src/index.ts:**
    Main server entry point that wires everything together:

    ```typescript
    import fastify from 'fastify';
    import cors from '@fastify/cors';
    import { initDb } from './db/index.js';
    import { registerFeedRoutes } from './routes/feed.js';
    import { registerScrapeRoutes } from './routes/scrape.js';
    import { registerHealthRoutes } from './routes/health.js';
    import { startScheduler } from './scheduler.js';
    import { RedditScraper } from './scrapers/reddit.js';
    import { getBtsScrapingConfig } from '@bts/shared/config/sources';

    const server = fastify({ logger: true });
    await server.register(cors, { origin: true }); // Allow all origins for dev

    // Initialize database
    const db = initDb();

    // Make db available to routes via decoration or closure
    // (pass db to route registration functions)

    // Register API routes under /api prefix
    await server.register(async (api) => {
      registerFeedRoutes(api, db);
      registerScrapeRoutes(api, db, scrapers);
      registerHealthRoutes(api, db);
    }, { prefix: '/api' });

    // Create scrapers
    const config = getBtsScrapingConfig();
    const scrapers = [new RedditScraper(config)];

    // Start hourly scheduler + initial scrape
    startScheduler(db, scrapers);

    // Start server
    const port = parseInt(process.env.PORT || '3001');
    await server.listen({ port, host: '0.0.0.0' });
    server.log.info(`Server running on port ${port}`);
    ```

    The order matters: initialize DB first, then register routes, then start scheduler, then listen.

    Add graceful shutdown: handle SIGINT/SIGTERM to close the Fastify server and stop the cron scheduler.
