- `errorStack: text('error_stack')` — full stack trace on failure, nullable

**ScrapedItem in `packages/server/src/scrapers/base.ts`:**

- `thumbnailUrl: string | null;` — set by each scraper during scrape
- `engagementStats: Record<string, number> | null;` — per-source typed JSON (e.g., `{upvotes: 42, comments: 5}` for Reddit, `{likes: 15}` for Bluesky, null for YouTube RSS/news/Tumblr)

Keep existing `score` and `commentCount` fields on ScrapedItem for backward compatibility with RedditScraper (it still populates them).

**Feed route updates in `packages/server/src/routes/feed.ts`:**

1. Add `isNull` to the drizzle-orm import: `import { and, desc, lt, eq, sql, isNull } from 'drizzle-orm';`
2. In the `/feed` GET handler, add `isNull(contentItems.deletedAt)` to the `conditions` array (always applied, not conditional).
3. In the total count query, add `isNull(contentItems.deletedAt)` to countWhere.
4. In both the `/feed` and `/feed/:id` response mapping, add:
   - `thumbnailUrl: row.thumbnailUrl ?? null`
   - `engagementStats: row.engagementStats ? JSON.parse(row.engagementStats) : null`
5. In `/feed/:id`, also add `isNull(contentItems.deletedAt)` to the WHERE clause so deleted items return 404.

**Orchestration updates in `packages/server/src/scrapers/base.ts`:**

1. In the upsert within `runAllScrapers()`, add to the `.values()` call:
   - `thumbnailUrl: item.thumbnailUrl`
   - `engagementStats: item.engagementStats ? JSON.stringify(item.engagementStats) : null`
2. In the `.onConflictDoUpdate()` set, add:
   - `engagementStats: sql\`excluded.engagement_stats\``
   - `thumbnailUrl: sql\`excluded.thumbnail_url\``
   (So engagement stats and thumbnails update on re-scrape)
3. **Change 30-day hard delete to 14-day soft delete:**
   Replace the existing cleanup block at the bottom of `runAllScrapers()`:
   - Change `thirtyDaysAgo` to `fourteenDaysAgo = new Date(Date.now() - 14 * 24 * 60 * 60 * 1000)`
   - Change `db.delete(contentItems)` to `db.update(contentItems).set({ deletedAt: new Date() })`
   - Add `isNull(contentItems.deletedAt)` to the WHERE clause so already-deleted items aren't re-touched
   - Use `lt(contentItems.publishedAt, fourteenDaysAgo)` (filter on publishedAt, not scrapedAt, so content that was scraped recently but published long ago still gets cleaned)
4. **Three-state run tracking:** In the scrape_runs update, set:
   - `duration: result.duration` in the success update
   - Status: if `result.items.length === 0`, set status to `'empty'` instead of `'success'`. If items found, keep `'success'`.
   - In the error catch block, also set `errorStack: error instanceof Error ? error.stack ?? String(error) : String(error)` and `duration: Date.now() - startedAt.getTime()`.
5. **Add `isNull` import** to the drizzle-orm import line at top.

**Data migration for existing Reddit items:**
Add a one-time migration in the orchestration: after the upsert loop (inside the try block, before updating scrape_runs to success), check if this is the Reddit scraper (`result.source === 'reddit'`). If so, run a single UPDATE to populate `engagement_stats` for any existing rows that have `score > 0` and `engagement_stats IS NULL`:
```sql
UPDATE content_items SET engagement_stats = json_object('upvotes', score, 'comments', comment_count) WHERE source = 'reddit' AND engagement_stats IS NULL AND score > 0
```
Use `db.run(sql\`...\`)` for this. This is idempotent — runs on every Reddit scrape but only affects rows without engagement_stats.
