<task type="auto">
  <name>Task 2: Add engagement-weighted ordering, deduplication, and feed filtering</name>
  <files>
    src/services/feeds.ts
  </files>
  <action>
    Modify `src/services/feeds.ts` to add engagement-weighted sorting, URL deduplication, 7-day age filtering, and enabled source filtering.

    **1. Enabled source filtering:**
    - At the start of both `fetchAllFeedsIncremental` and `fetchAllFeeds`, filter `config.sources` to only include sources where `source.enabled !== false` (treat undefined/missing as enabled):
      ```typescript
      const enabledSources = config.sources.filter((s) => s.enabled !== false);
      ```
    - Use `enabledSources` instead of `config.sources` for the mapping.

    **2. Age filtering (7 days max):**
    - Create helper: `const MAX_AGE_MS = 7 * 24 * 60 * 60 * 1000;`
    - After collecting all items, filter out posts older than 7 days:
      ```typescript
      const now = Date.now();
      const fresh = items.filter(item => now - item.timestamp < MAX_AGE_MS);
      ```

    **3. URL deduplication:**
    - Create `normalizeUrl(url: string): string` helper:
      - Parse URL, remove utm_source, utm_medium, utm_campaign, ref params
      - Return normalized string. On parse error, return original.
    - Create `deduplicateByUrl(items: FeedItem[]): FeedItem[]`:
      - Build Map<normalizedUrl, FeedItem>
      - For duplicates, keep the item with higher engagement (use `getEngagementValue`)
      - Return Map values as array

    **4. Engagement-weighted scoring:**
    - Create `getEngagementValue(item: FeedItem): number`:
      - If no stats, return 0
      - Reddit: `(stats.upvotes ?? 0) + (stats.comments ?? 0) * 2`
      - YouTube: `(stats.views ?? 0) / 100 + (stats.likes ?? 0)`
      - Tumblr: `stats.notes ?? 0`
      - Default: 0
    - Create `computeFeedScore(item: FeedItem, now: number): number`:
      - Recency score: `Math.max(0, 1 - (now - item.timestamp) / MAX_AGE_MS)` (1.0 = just posted, 0.0 = 7 days old)
      - Engagement score: `engagement > 0 ? Math.log10(1 + engagement) / 6 : 0` (log scale, normalized to ~0-1)
      - Combined: `0.5 * recencyScore + 0.5 * engagementScore` (equal weight per locked decision)

    **5. Apply to sort:**
    - Replace the existing `.sort((a, b) => b.timestamp - a.timestamp)` in both functions with the pipeline:
      1. Filter age (7 days)
      2. Deduplicate by URL
      3. Sort by `computeFeedScore` descending
    - In `fetchAllFeedsIncremental`, the `onItems` callback should also apply the full pipeline (age filter + dedup + engagement sort) so the progressive UI updates show the smart ordering.

    **6. Per-source-type cap (Claude's discretion):**
    - After dedup but before sorting, apply a soft cap: max 30 items per source type. This prevents Reddit (with 6+ subreddits) from producing 60+ items that crowd out other types.
    - Group items by `item.source`, sort each group by engagement descending, keep top 30 per group, then merge and sort by combined score.

    Keep all helper functions (getEngagementValue, computeFeedScore, normalizeUrl, deduplicateByUrl) as module-private (not exported). They're internal to feeds.ts.
  </action>

  </verify>
