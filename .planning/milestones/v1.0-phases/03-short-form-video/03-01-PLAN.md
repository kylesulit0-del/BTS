<task type="auto">
  <name>Task 1: Create video URL detection utility and extend FeedItem type</name>
  <files>src/utils/videoDetect.ts, src/types/feed.ts</files>
  <action>
Create `src/utils/videoDetect.ts` with:

1. Export `VideoType` type: `'youtube-short' | 'tiktok'`
2. Export `VideoInfo` interface: `{ videoType: VideoType; videoId: string; isShortUrl?: boolean }`
3. Export `detectVideo(url: string, selftext?: string): VideoInfo | null` function that:
   - Concatenates `url` and `selftext` into a single search string
   - Checks YouTube Shorts pattern first: `(?:https?:\/\/)?(?:www\.|m\.)?youtube\.com\/shorts\/([a-zA-Z0-9_-]{11})` -- extracts 11-char video ID
   - Checks TikTok full URL: `(?:https?:\/\/)?(?:www\.)?tiktok\.com\/@[\w.-]+\/video\/(\d+)` -- extracts numeric video ID, `isShortUrl: false`
   - Checks TikTok short URLs: `(?:https?:\/\/)?(?:vm|vt)\.tiktok\.com\/([A-Za-z0-9]+)\/?` -- stores the matched short URL path as ID, sets `isShortUrl: true` (cannot resolve client-side without CORS proxy)
   - Returns `null` if no match

Extend `src/types/feed.ts`:
- Add `export type VideoType = 'youtube-short' | 'tiktok';` (import from videoDetect.ts or define here -- prefer defining here alongside FeedItem for colocation)
- Add optional fields to `FeedItem`: `videoType?: VideoType` and `videoId?: string`
  </action>
  <verify>
Run `npx tsc --noEmit` -- no type errors. Verify videoDetect.ts exports are importable from reddit.ts path.
  </verify>
  <done>
videoDetect.ts exists with detectVideo function that correctly identifies YouTube Shorts and TikTok URLs in all formats. FeedItem type has optional videoType and videoId fields. TypeScript compiles cleanly.
  </done>
