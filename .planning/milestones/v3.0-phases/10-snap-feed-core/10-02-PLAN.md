    **SeeMoreSheet.tsx** — Bottom sheet overlay component per user decision:

    Props: `{ text: string; isOpen: boolean; onClose: () => void }`

    1. **Backdrop:** Fixed position, full screen, semi-transparent black (`rgba(0, 0, 0, 0.5)`), z-index 200. Tap on backdrop calls `onClose`.

    2. **Sheet panel:**
       - Fixed position, bottom: 0, left: 0, right: 0
       - Max height: 80vh (per user decision: ~80% of screen)
       - Background: `var(--bg-card, #1a1a2e)`
       - Border radius: 16px 16px 0 0 (rounded top corners)
       - z-index: 201
       - Transform: `translateY(100%)` when closed, `translateY(0)` when open
       - Transition: `transform 0.3s cubic-bezier(0.32, 0.72, 0, 1)` (iOS-like easing per research recommendation)
       - Overflow-y: auto (scrollable if content exceeds 80vh)
       - Padding: 24px 16px

    3. **Handle bar:** Small horizontal bar at top (36px wide, 4px tall, rounded, subtle gray) indicating drag affordance.

    4. **Swipe-down to dismiss:**
       - Track touchstart Y position on the sheet panel
       - On touchmove, if dragging downward, apply translateY transform proportional to drag distance
       - On touchend, if drag distance > 80px (threshold), call onClose. Otherwise, spring back.
       - Use the same `cubic-bezier(0.32, 0.72, 0, 1)` easing for spring-back animation.

    5. **Content:** Render the full text as a paragraph. No extra metadata or images (per user decision: "Full text only").

    6. **Portal rendering:** Render via `createPortal` to `document.body` to escape scroll container stacking context.

    **Wire truncation into SnapCardImage.tsx and SnapCardText.tsx:**

    1. **Text truncation:** For `item.preview` text in both image and text cards:
       - Apply CSS `-webkit-line-clamp: 5` to the preview/body text element
       - Use a ref on the text element to detect overflow: `scrollHeight > clientHeight`
       - If overflowing, append "...See More" inline text link after the clamped text
       - The "See More" tap target calls a callback that opens the SeeMoreSheet

    2. **State management:** Each card manages its own `seeMoreOpen` boolean state. When "See More" is tapped, open the SeeMoreSheet with the full `item.preview` text.

    3. **Overflow detection hook:** Create a small inline helper or use a `useEffect` + `useRef` pattern:
       ```
       const textRef = useRef<HTMLParagraphElement>(null);
       const [isOverflowing, setIsOverflowing] = useState(false);
       useEffect(() => {
         const el = textRef.current;
         if (el) setIsOverflowing(el.scrollHeight > el.clientHeight);
       }, [item.preview]);
       ```
