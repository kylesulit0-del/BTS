**Create thumbnail utilities at `packages/server/src/scrapers/thumbnail.ts`:**

```typescript
import { fetchWithRetry } from './utils.js';
```

Implement these exported functions:

1. `extractOgImage(articleUrl: string): Promise<string | null>` — Fetch article HTML (use fetchWithRetry), extract only the `<head>` section (find `</head>` and slice), then regex-match `<meta property="og:image" content="URL">` or `<meta content="URL" property="og:image">` (both attribute orders). If found, check against `isKnownLogo()`. If it's a logo, fall back to `extractFirstArticleImage()`. Return the URL or null.

2. `extractFirstArticleImage(html: string): string | null` — Regex match the first `<img` tag's `src` attribute in the HTML body. Skip images smaller than 100px (if width/height attributes are present and < 100). Return URL or null.

3. `isKnownLogo(url: string): boolean` — Check URL against patterns: `/logo`, `/brand`, `/default-og`, `/favicon`, `/icon`, `site-logo`, `default-image`, `og-default`. Case-insensitive match. Return true if URL matches any pattern.

4. `validateThumbnail(url: string): Promise<boolean>` — Send HEAD request with 3-second timeout (use AbortSignal.timeout(3000)). Return true if response is OK and content-type starts with `image/`. Return false on any error. Skip validation for known CDN patterns (youtube.com/vi/, tumblr.com, bsky.app).

5. `extractRssThumbnail(item: any): string | null` — Extract image from RSS feed item. Check `item.enclosure?.url` (if type starts with `image/`), then `item['media:content']?.$.url`, then `item['media:thumbnail']?.$.url`. Return first match or null.

```typescript
import type { Scraper, ScrapedItem, ScraperResult } from './base.js';
import type { GroupScrapingConfig } from '@bts/shared/config/sources.js';
import Parser from 'rss-parser';
import { delay } from './utils.js';
```

- `name = 'youtube'`
- Constructor takes `GroupScrapingConfig`
- Create `Parser` instance with customFields: `item: [['yt:videoId', 'videoId']]`
- In `scrape()`:
  1. Filter config.sources for `type === 'youtube'` and `enabled !== false`, sorted by priority
  2. For each source, build feed URL: `https://www.youtube.com/feeds/videos.xml?channel_id=${source.url}`
  3. Parse with `parser.parseURL(feedUrl)` wrapped in try/catch
  4. For each feed item, extract:
     - `videoId` from the custom field `(item as any).videoId`
     - `title` from `item.title`
     - `url` as `https://www.youtube.com/watch?v=${videoId}` (or `item.link`)
     - `publishedAt` from `item.isoDate`
     - `thumbnailUrl` as `https://img.youtube.com/vi/${videoId}/mqdefault.jpg` (320x180 per user decision — do NOT parse media:thumbnail, it has a known bug per research pitfall 1)
  5. Apply keyword filtering if `source.needsFilter` is true (same pattern as RedditScraper — check if title contains any keyword case-insensitively)
  6. Map to `ScrapedItem[]` with:
     - `externalId: 'yt-' + videoId`
     - `source: 'youtube'`
     - `sourceDetail: source.label`
     - `score: 0`, `commentCount: 0` (YouTube RSS provides no engagement — per user decision, YouTube Data API deferred)
     - `engagementStats: null` (no stats from RSS feed)
     - `thumbnailUrl: 'https://img.youtube.com/vi/${videoId}/mqdefault.jpg'`
     - `flair: null`, `contentType: null`
  7. Add 1-second delay between channels to be respectful
  8. Build `ScraperResult` per source with items, source, sourceDetail, duration, errors

```typescript
import type { Scraper, ScrapedItem, ScraperResult } from './base.js';
import type { GroupScrapingConfig } from '@bts/shared/config/sources.js';
import Parser from 'rss-parser';
import { delay } from './utils.js';
import { extractOgImage, extractRssThumbnail, validateThumbnail } from './thumbnail.js';
```

- `name = 'rss'`
- Constructor takes `GroupScrapingConfig`
- Create `Parser` instance (default config, no custom fields needed for RSS 2.0)
- In `scrape()`:
  1. Filter config.sources for `type === 'rss'` and `enabled !== false`, sorted by priority
  2. For each source, parse RSS feed: `parser.parseURL(source.url)` with try/catch
  3. Limit items to `source.fetchCount`
  4. Apply keyword filtering if `source.needsFilter` (check title + contentSnippet against config.keywords)
  5. For each item, extract thumbnail using progressive strategy:
     a. Try `extractRssThumbnail(item)` (checks enclosure, media:content, media:thumbnail)
     b. If no thumbnail from RSS, try `extractOgImage(item.link)` (fetches article HTML)
     c. If a thumbnail was found, validate with `validateThumbnail()` — if invalid, set to null
     d. Use `Promise.allSettled()` for batch thumbnail validation (batch all items' thumbnails, not sequential)
  6. Map to `ScrapedItem[]` with:
     - `externalId: 'rss-' + Buffer.from(item.link || item.guid || '').toString('base64url').slice(0, 32)` (stable ID from URL)
     - `url: item.link`
     - `title: item.title`
     - `source: 'rss'`
     - `sourceDetail: source.label`
     - `score: 0`, `commentCount: 0`
     - `engagementStats: null` (news sites have no engagement stats)
     - `thumbnailUrl: extractedThumbnail` (from progressive strategy above)
     - `flair: null`, `contentType: null`
     - `publishedAt: new Date(item.isoDate || item.pubDate || Date.now())`
  7. Add 1-second delay between news sites
  8. Build `ScraperResult` per source
