```typescript
import type { Scraper, ScrapedItem, ScraperResult } from './base.js';
import type { GroupScrapingConfig } from '@bts/shared/config/sources.js';
import Parser from 'rss-parser';
import { delay } from './utils.js';
```

- `name = 'tumblr'`
- Constructor takes `GroupScrapingConfig`
- Create `Parser` instance (default config — Tumblr uses standard RSS 2.0)
- In `scrape()`:
  1. Filter config.sources for `type === 'tumblr'` and `enabled !== false`, sorted by priority
  2. For each source, parse RSS: `parser.parseURL(source.url)` with try/catch
  3. Limit items to `source.fetchCount`
  4. Apply keyword filtering if `source.needsFilter` (check title + content against config.keywords, case-insensitive)
  5. For each item, extract thumbnail:
     - Tumblr puts images in `item.content` or `item['content:encoded']` as HTML `<img>` tags inside CDATA
     - Extract first image via regex: `/<img[^>]+src=["']([^"']+)["']/i`
     - This is the first image only per user decision ("Tumblr: first image from the post only")
     - No HEAD validation needed for Tumblr CDN URLs (known reliable CDN — `*.media.tumblr.com`)
  6. Map to `ScrapedItem[]`:
     - `externalId: 'tumblr-' + Buffer.from(item.link || item.guid || '').toString('base64url').slice(0, 32)`
     - `url: item.link`
     - `title: item.title || '(untitled)'` (Tumblr posts can be untitled — use a fallback)
     - `source: 'tumblr'`
     - `sourceDetail: source.label`
     - `score: 0`, `commentCount: 0`
     - `engagementStats: null` (Tumblr RSS provides no engagement stats)
     - `thumbnailUrl`: extracted first image or null
     - `flair: null`, `contentType: null`
     - `publishedAt: new Date(item.isoDate || item.pubDate || Date.now())`
  7. Add 1-second delay between blogs
  8. Build `ScraperResult` per source

**Error handling:** If a Tumblr blog's RSS feed returns 404 or an error, log a warning and push an empty result with the error message. Do not crash the entire scraper.

**Create `packages/server/src/scrapers/bluesky.ts`:**

Implement `BlueskyScraper` class implementing the `Scraper` interface:

```typescript
import type { Scraper, ScrapedItem, ScraperResult } from './base.js';
import type { GroupScrapingConfig } from '@bts/shared/config/sources.js';
import { BskyAgent } from '@atproto/api';
import { delay } from './utils.js';
```

- `name = 'bluesky'`
- Constructor takes `GroupScrapingConfig`
- Create `BskyAgent` instance: `new BskyAgent({ service: 'https://bsky.social' })`
- In `scrape()`:
  1. **Auth check:** If `BLUESKY_HANDLE` or `BLUESKY_APP_PASSWORD` env vars are missing, log a warning and return an empty result with an error message: `'Bluesky credentials not configured (BLUESKY_HANDLE, BLUESKY_APP_PASSWORD)'`. Do NOT throw — graceful degradation.
  2. **Login:** `await agent.login({ identifier: process.env.BLUESKY_HANDLE!, password: process.env.BLUESKY_APP_PASSWORD! })`. Wrap in try/catch — if auth fails, return empty result with error.
  3. Filter config.sources for `type === 'bluesky'` and `enabled !== false`
  4. For each source, the `url` field contains the search keyword (e.g., `'BTS'`). Also use the top 5 keywords from `config.keywords` for broader search.
  5. Search posts: `agent.app.bsky.feed.searchPosts({ q: keyword, limit: source.fetchCount, sort: 'latest' })`
  6. For each post in `response.data.posts`:
     - `text = (post.record as any).text` (post text content)
     - `likes = post.likeCount ?? 0` (per user decision: capture likes only, not reposts)
     - `uri = post.uri` (at:// URI)
     - Build web URL: `https://bsky.app/profile/${post.author.handle}/post/${post.uri.split('/').pop()}`
     - `createdAt = (post.record as any).createdAt` (ISO timestamp)
     - Extract embedded image: `post.embed?.images?.[0]?.thumb ?? null` (if embed type is images)
       - Check `post.embed?.$type === 'app.bsky.embed.images#view'` before accessing images
       - Also check `post.embed?.$type === 'app.bsky.embed.recordWithMedia#view'` for quote posts with images
  7. Map to `ScrapedItem[]`:
     - `externalId: 'bsky-' + post.uri.split('/').pop()` (rkey from AT URI)
     - `url`: constructed web URL
     - `title: text.slice(0, 200)` (Bluesky posts are title-less — use first 200 chars of text as title)
     - `source: 'bluesky'`
     - `sourceDetail: 'Bluesky'`
     - `score: 0`, `commentCount: 0` (legacy fields, not used for Bluesky)
     - `engagementStats: { likes }` (per user decision: likes only)
     - `thumbnailUrl`: embedded image thumb or null
     - `flair: null`, `contentType: null`
     - `publishedAt: new Date(createdAt)`
  8. Deduplicate posts across keyword searches (same URI = same post) before returning
  9. Add 500ms delay between keyword searches to respect rate limits (Bluesky rate limit: 3000 req/5min for search)
  10. Build `ScraperResult` with all items from all keyword searches combined
