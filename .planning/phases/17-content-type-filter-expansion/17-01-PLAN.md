<objective>
Expand the content type taxonomy across shared types, LLM pipeline, and server to support 8 content types (news, fan_art, fan_fiction, music, discussion, social_posts, media, general) with source-level defaults that skip LLM classification for known sources.

Purpose: The current 7-type taxonomy (news, fan_art, meme, video, discussion, translation, official) doesn't cover fan fiction or music content, and forces all items through LLM even when the source type is unambiguous. Expanding the taxonomy and adding source-level defaults improves classification accuracy and reduces LLM costs.

Output: Updated shared types, LLM schemas/prompt, and pipeline with source-level defaults. Server ?contentType= filter works for all new values.
</objective>

**2. Split raw items into two groups in `runPipeline`.**

After selecting raw items (the `rawItems` query), before marking as 'pending', partition items:

```typescript
// Partition: items with source-level defaults vs items needing LLM
const defaultItems: typeof rawItems = [];
const llmItems: typeof rawItems = [];

for (const item of rawItems) {
  const defaultType = SOURCE_DEFAULT_CONTENT_TYPES.get(item.source);
  if (defaultType) {
    defaultItems.push(item);
  } else {
    llmItems.push(item);
  }
}

// Apply source-level defaults immediately (skip LLM)
for (const item of defaultItems) {
  const defaultType = SOURCE_DEFAULT_CONTENT_TYPES.get(item.source)!;
  db.update(contentItems)
    .set({
      moderationStatus: 'approved',
      contentType: defaultType,
      moderatedAt: new Date(),
    })
    .where(eq(contentItems.id, item.id))
    .run();
}

if (defaultItems.length > 0) {
  console.log(`[pipeline] Source defaults applied: ${defaultItems.length} items (${[...new Set(defaultItems.map(i => i.source))].join(', ')})`);
}
```

Then only claim and process `llmItems` through LLM (replace the old `rawItems` usage for pending/batching with `llmItems`). If `llmItems.length === 0`, skip LLM entirely and still create the pipeline run record.
