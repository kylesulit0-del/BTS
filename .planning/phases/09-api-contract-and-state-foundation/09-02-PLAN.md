    1. Create `packages/frontend/src/hooks/useFeedState.ts`:
       - Import `useReducer`, `useEffect` from 'react' and `useSearchParams` from 'react-router-dom'.
       - Define and export `SortMode` type: `'recommended' | 'newest' | 'oldest' | 'popular' | 'discussed'`
       - Define and export `FeedState` interface: `{ sort: SortMode; source: string; contentType: string }`
         - `source` defaults to `'all'` (no filter). `contentType` defaults to `'all'`.
         - Do NOT include `members` in FeedState -- member/bias filtering stays in localStorage via existing `useBias` hook (per research anti-pattern: member selections are personal preferences, not shareable URL state).
       - Define `FeedAction` discriminated union:
         - `{ type: 'SET_SORT'; sort: SortMode }`
         - `{ type: 'SET_SOURCE'; source: string }`
         - `{ type: 'SET_CONTENT_TYPE'; contentType: string }`
         - `{ type: 'RESET' }`
       - Implement `feedReducer(state: FeedState, action: FeedAction): FeedState` as a pure switch/case.
       - Implement `useFeedState()` hook:
         a. Call `const [searchParams, setSearchParams] = useSearchParams();`
         b. Initialize state from URL params on first render:
            - `sort`: read `searchParams.get('sort')`, validate against `VALID_SORTS` array, default to `'recommended'`.
            - `source`: read `searchParams.get('source')` or default `'all'`.
            - `contentType`: read `searchParams.get('type')` or default `'all'`.
         c. Call `useReducer(feedReducer, initialState)`.
         d. Add a `useEffect` that syncs state TO URL params. CRITICAL: depend on `state.sort`, `state.source`, `state.contentType` (individual values), NOT on `searchParams` (avoids infinite re-render loop per research pitfall #1). Use `setSearchParams(params, { replace: true })` to avoid polluting browser history (per research pitfall #2). Only set non-default values (omit sort=recommended, source=all, type=all from URL).
         e. Return `[state, dispatch] as const`.

    2. Modify `packages/frontend/src/hooks/useFeed.ts`:
       - Change the function signature from `useFeed(filter: FeedSource | "all", biases: BiasId[])` to `useFeed(feedState: { sort: string; source: string; contentType: string }, biases: BiasId[])`.
       - Inside `load()`, pass `sort` to `fetchFeed()`: `await fetchFeed({ force, onItems, source: apiMode && feedState.source !== 'all' ? feedState.source : undefined, sort: feedState.sort })`.
       - Update the local source filtering to use `feedState.source` instead of `filter`.
       - Remove the old `filter` parameter usage. The `contentType` from feedState is NOT passed to fetchFeed server-side -- it stays as client-side filtering in News.tsx (consistent with existing pattern where contentType filtering is local).
       - Add `feedState.sort` and `feedState.source` to the `useCallback` dependency array for `load` and the `useEffect` dependency.

    3. Modify `packages/frontend/src/pages/News.tsx`:
       - Import `useFeedState` from `../hooks/useFeedState`.
       - Replace the separate `useState` calls for `filter` and `contentTypeFilter` with a single `const [feedState, dispatch] = useFeedState();`.
       - Pass `feedState` to `useFeed`: `const { items: rawItems, ... } = useFeed(feedState, biases);`.
       - Update content type filtering to use `feedState.contentType` instead of `contentTypeFilter`.
       - Update `FeedFilter` onChange to dispatch: `onChange={(source) => dispatch({ type: 'SET_SOURCE', source })}`, and pass `active={feedState.source}`.
       - Update content type pill clicks to dispatch: `onClick={() => dispatch({ type: 'SET_CONTENT_TYPE', contentType: type })}`.
       - Update the active check for content type pills to use `feedState.contentType`.
       - Remove the old `useState` for `filter` and `contentTypeFilter` -- they are now in the reducer.
       - Keep `viewMode` as local `useState` (list/swipe toggle is UI preference, not feed state).
  </action>
