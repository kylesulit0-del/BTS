  <action>
    **snap.ts** — Create configurable constants file:
    - `DOM_WINDOW_SIZE = 5` (number of items rendered in DOM at any time)
    - Add a comment documenting how to adjust the value
    - Export as named constants

    **useSnapFeed.ts** — Create the core snap feed hook accepting `items: FeedItem[]` and returning `{ windowedItems, currentIndex, containerRef }`:

    1. **State:** `currentIndex` (number) tracked via useState, initialized to 0.

    2. **DOM windowing via `getWindowedItems()`:**
       - Given `items[]` and `currentIndex`, return an array of `{ item: FeedItem, realIndex: number }[]` of length `DOM_WINDOW_SIZE`
       - Use modular arithmetic for seamless wrapping: `((currentIndex + offset) % len + len) % len`
       - The offset range is `-Math.floor(DOM_WINDOW_SIZE / 2)` to `+Math.floor(DOM_WINDOW_SIZE / 2)`
       - **Edge case:** When `items.length <= DOM_WINDOW_SIZE`, skip windowing entirely — return all items with their real indices. No modular wrapping to avoid duplicate keys.
       - **Edge case:** When `items.length === 0`, return empty array.

    3. **IntersectionObserver for snap detection:**
       - Create observer on `containerRef` (the scroll container) with `{ root: containerRef.current, threshold: 0.6 }`
       - Each card gets a `data-realindex` attribute with its real item index
       - When a card crosses the 0.6 threshold, update `currentIndex` to its `data-realindex` value
       - Guard: only accept index if the element still exists in the current render (prevent stale callbacks)
       - Re-create observer when `windowedItems` changes (dependency of useEffect)

    4. **Scroll position management for window shifts:**
       - When `currentIndex` changes and the DOM window shifts, the scroll position needs adjustment to prevent jumps
       - Since all cards are fixed height (100svh), the math is deterministic
       - Use `useLayoutEffect` to adjust `scrollTop` synchronously when the window composition changes
       - Track the previous window's center index to calculate the delta

    5. **Seamless looping:**
       - The modular arithmetic in `getWindowedItems` handles looping implicitly
       - No explicit scroll reset needed — the window just wraps around
       - When `items.length` is small (< DOM_WINDOW_SIZE), render all items; the scroll container handles natural looping via scroll position reset when reaching the end (scrollTop = 0 when at bottom)

    Types: Export `WindowedItem` type: `{ item: FeedItem; realIndex: number }`
  </action>

    - useSnapFeed.ts exports useSnapFeed hook and WindowedItem type
    - getWindowedItems correctly handles: normal case (10 items), edge case (3 items < window size), edge case (0 items)
  </verify>
