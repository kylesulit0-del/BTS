    **useSnapVideo.ts** ‚Äî Video lifecycle hook for snap feed:

    This hook manages video playback in the snap feed context, enforcing single-iframe and autoplay/pause behavior.

    1. **Session mute state** (module-level, not React state):
       - `let sessionMuted: boolean = true` ‚Äî starts muted per browser autoplay policy
       - `getSessionMuted(): boolean` and `setSessionMuted(muted: boolean): void` exports
       - After user's first unmute action, `sessionMuted` becomes false
       - All subsequent videos auto-start with the user's last mute preference
       - NOTE: Research says "sound on by default" is physically impossible per browser policy. Start muted, auto-unmute after first user interaction. This matches TikTok's actual behavior.

    2. **Hook: `useSnapVideo(isActive: boolean, iframeRef: RefObject<HTMLIFrameElement | null>, videoType: VideoType)`:**
       Returns `{ muted: boolean, toggleMute: () => void, togglePlayPause: () => void, isPlaying: boolean }`

       - **When `isActive` becomes true:**
         - If iframe exists, send play command via `sendCommand` from existing useVideoAutoplay.ts
         - Apply current session mute state: if `sessionMuted`, send mute; else send unmute
         - Update module-level `currentlyPlaying` to this iframe (reuse pattern from useVideoAutoplay)
         - Set `isPlaying = true`

       - **When `isActive` becomes false:**
         - If iframe exists, send pause command
         - If this was `currentlyPlaying`, clear it
         - Set `isPlaying = false`

       - **toggleMute():**
         - Toggle `muted` state
         - Send mute/unmute command to iframe via `sendMuteCommand` from useVideoAutoplay
         - Update `sessionMuted` via `setSessionMuted(!muted)` ‚Äî this persists for subsequent videos

       - **togglePlayPause():**
         - If playing, send pause; if paused, send play
         - Toggle `isPlaying`

    3. **Import sendCommand and sendMuteCommand** from `../hooks/useVideoAutoplay`. These are already exported. Do NOT duplicate the postMessage logic.

  <action>
    **useSwipeGesture.ts** ‚Äî Horizontal swipe gesture detection hook:

    `useSwipeGesture(onSwipeRight: () => void, sourceColor?: string)` returns `{ handlers: TouchEventHandlers, style: CSSProperties, swiping: boolean }`

    1. **Touch tracking refs:**
       - `startX`, `startY` (touch start position)
       - `deltaX` (current horizontal displacement)
       - `isSwiping` (boolean: has the gesture been claimed as horizontal?)

    2. **onTouchStart:** Record start position. Reset deltaX and isSwiping.

    3. **onTouchMove:**
       - Calculate deltaX (current X - startX) and deltaY (current Y - startY)
       - **Axis locking:** If not yet claimed and `|deltaX| > 10` (dead zone):
         - If `|deltaX| > |deltaY| * 1.5` ‚Üí claim as horizontal swipe, set `isSwiping = true`
         - Else ‚Üí vertical scroll dominates, do NOT claim
       - If `isSwiping` and `deltaX > 0` (right direction only, per user decision: left-swipe does nothing):
         - Update the `deltaX` ref for visual transform
         - Set CSS transform: `translateX(${deltaX}px)` on the card
       - If `isSwiping` and `deltaX <= 0`: clamp to 0 (no left-swipe visual)
       - **CRITICAL:** Never call `preventDefault()` on vertical-dominant touches ‚Äî this would break scroll-snap.

    4. **onTouchEnd:**
       - If `isSwiping` and `deltaX > threshold` (120px per research recommendation):
         - Trigger `onSwipeRight()` callback (opens source link)
         - Animate card fully off-screen to the right before resetting
       - If `isSwiping` and `deltaX <= threshold`:
         - Spring back: animate card to `translateX(0)` with `cubic-bezier(0.32, 0.72, 0, 1)` easing
       - Reset all tracking state

    5. **Return value:**
       - `handlers`: `{ onTouchStart, onTouchMove, onTouchEnd }` ‚Äî React touch event handlers
       - `style`: `{ transform, transition }` CSS properties to apply to the card element
       - `swiping`: boolean indicating if a swipe is in progress (for showing background color)

    **Wire into SnapCard.tsx:**

    1. Import `useSwipeGesture` hook.

    2. Create the swipe callback: `() => window.open(item.url, '_blank', 'noopener')`

    3. Get source color for the card's source (from `sourceBadgeColors` map ‚Äî reddit orange, youtube red, etc.)

    4. Call `useSwipeGesture(openUrl, sourceColor)` to get handlers and style.

    5. Apply `handlers` (onTouchStart, onTouchMove, onTouchEnd) to the snap card wrapper div.

    6. Apply `style` (transform, transition) to the snap card content div (the inner element that slides).

    7. **Background reveal:** When `swiping` is true, show the source-colored background behind the sliding card content:
       - The outer wrapper has `background: sourceColor` (or fallback `var(--theme-primary)`)
       - The inner card content slides right, revealing the colored background underneath
       - No URL preview during slide (per user decision)

    **App.css** ‚Äî Add swipe gesture styles:
    - `.snap-card-swipe-bg` ‚Äî colored background revealed during swipe
    - `.snap-card-content` ‚Äî the slideable inner element
    - Transition for spring-back: `transform 0.3s cubic-bezier(0.32, 0.72, 0, 1)`
