**FilterSheet.tsx** -- New component following SeeMoreSheet portal pattern:

1. Props interface:
   ```typescript
   interface FilterSheetProps {
     isOpen: boolean;
     onClose: () => void;
     feedState: FeedState;
     dispatch: React.Dispatch<FeedAction>;
   }
   ```

2. Internal tab state: `activeTab: "source" | "member" | "type"` defaulting to `"source"`.

3. Tab header row inside the sheet:
   - Three tab buttons: "Source", "Member", "Type"
   - Active tab gets highlighted styling (underline or filled background)
   - Tapping a tab switches the displayed chip grid

4. **Source tab content:**
   - Read available sources from config: `getConfig().sources` to get enabled source types
   - Display labels from `config.labels.sourceLabels`
   - Show as tappable chips: Reddit, YouTube, RSS, Tumblr, Bluesky
   - Active (selected) chips get highlighted background (theme-primary with opacity)
   - Tapping dispatches `TOGGLE_SOURCE` with the source type string (e.g., "reddit", "youtube")

5. **Member tab content:**
   - Read members from config: `getConfig().members`
   - Show as tappable chips with emoji prefix: e.g., "RM", "Jin", "Suga", etc.
   - Display OT7 as a special chip. Treat OT7 as group content -- use the member ID convention. If the config has an OT7 member entry, use its ID. Otherwise, add "ot7" as a special filter value handled separately.
   - Active chips highlighted
   - Tapping dispatches `TOGGLE_MEMBER` with the member ID string (e.g., "rm", "jin")

6. **Type tab content:**
   - Display the 4 user-facing categories as chips: Video, Image, News, Discussion
   - These are NOT the raw ContentType values. Define a constant mapping:
     ```typescript
     const CONTENT_TYPE_CATEGORIES = [
       { id: "video", label: "Video" },
       { id: "image", label: "Image" },
       { id: "news", label: "News" },
       { id: "discussion", label: "Discussion" },
     ];
     ```
   - Tapping dispatches `TOGGLE_CONTENT_TYPE` with the category ID (e.g., "video", "image")
   - The mapping from category to actual ContentType values is handled in News.tsx filtering (Plan 01 Task 2 already set this up)

7. **"Clear all" button** at the bottom of the sheet:
   - Only visible when any filters are active
   - Dispatches `CLEAR_ALL_FILTERS`

8. **Portal rendering** via `createPortal(content, document.body)` -- same as SeeMoreSheet.

9. **Swipe-to-dismiss** -- copy the SeeMoreSheet pattern:
   - `touchStartY` ref, `currentTranslateY` ref
   - `onTouchStart`: capture start Y
   - `onTouchMove`: if deltaY > 0 (swiping down), translate sheet downward. Check if the sheet's inner scroll is at top before allowing drag (to avoid blocking scrollable content).
   - `onTouchEnd`: if `currentTranslateY > 80`, call `onClose()`. Otherwise spring back to 0.
   - Use `transition: transform 0.3s cubic-bezier(0.32, 0.72, 0, 1)` for spring animation.

10. **Backdrop tap-to-dismiss**: outer `div` with `onClick={onClose}`, inner sheet with `e.stopPropagation()`.

11. **Body scroll lock**: when `isOpen`, set `document.body.style.overflow = "hidden"`. Restore on close/unmount.

12. **Gesture conflict prevention**: when the filter sheet is open, the `useVerticalPaging` hook in SnapFeed should be disabled. This is handled by passing `enabled: false` -- Plan 01 should have set up the wiring for this in News.tsx. If not, add an `isFilterOpen` state in News.tsx that gets passed to SnapFeed as a prop to disable paging. The portal rendering already isolates touch events, but the backdrop could bubble. The `e.stopPropagation()` on the sheet content prevents this.
