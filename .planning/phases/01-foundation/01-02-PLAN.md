<task type="auto">
  <name>Task 1: Create config type definitions</name>
  <files>
    src/config/types.ts
  </files>
  <action>
Create `src/config/types.ts` with the following interfaces (per RESEARCH.md Pattern 4, refined by user decisions in CONTEXT.md):

1. **MemberConfig** interface:
   - id: string
   - stageName: string
   - realName: string
   - aliases: string[] (all searchable names/nicknames/misspellings for keyword matching)
   - emoji: string
   - color: string
   - birthday: string
   - role: string
   - position: string
   - image: string
   - gallery?: string[]
   - bio: string
   - funFacts: string[]
   - soloProjects: string[]
   - socialMedia: { platform: string; handle: string; url?: string }[]

2. **SourceEntry** interface:
   - type: string (e.g., "reddit", "youtube", "rss", "twitter")
   - id: string (unique identifier for this source instance)
   - label: string (display name, e.g., "r/bangtan", "BANGTANTV")
   - url: string (base URL or identifier -- subreddit name, channel ID, RSS URL)
   - needsFilter: boolean (whether to apply keyword filtering)
   - fetchCount?: number (max items to fetch)
   - refreshInterval?: number (override default refresh interval in ms)
   - priority?: number (display priority, lower = higher)

3. **ThemeConfig** interface:
   - groupName: string (e.g., "BTS")
   - groupNameNative: string (e.g., "방탄소년단")
   - tagline: string (e.g., "Beyond The Scene · ARMY Forever")
   - fandomName: string (e.g., "ARMY")
   - primaryColor: string (hex, e.g., "#562B8B")
   - accentColor: string (hex)
   - darkColor: string (hex)
   - logoUrl: string (path to group photo)
   - socialLinks: { platform: string; handle: string; url: string }[]

4. **GroupConfig** interface:
   - members: MemberConfig[]
   - sources: SourceEntry[]
   - theme: ThemeConfig
   - keywords: RegExp (compiled from member aliases + group terms)

Per user decision: TypeScript types serve as documentation. No special template role for BTS config.
  </action>
  <verify>
    - `npx tsc --noEmit` compiles without errors
    - `src/config/types.ts` exports GroupConfig, MemberConfig, SourceEntry, ThemeConfig
  </verify>
  <done>
    Config type hierarchy is defined. GroupConfig composes MemberConfig[], SourceEntry[], ThemeConfig, and a compiled keywords RegExp. Types are the canonical interface for all group-specific data.
  </done>
