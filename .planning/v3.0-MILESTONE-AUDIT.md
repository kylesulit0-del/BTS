## Cross-Phase Integration

**Connected exports:** 19/22
**Issues found:** 3

### INT-01: windowedItems dead code (SNAP-04) — medium

`useSnapFeed.ts` computes a 5-item `windowedItems` array using `DOM_WINDOW_SIZE=5` from `snap.ts`, but `SnapFeed.tsx` renders `visibleItems` (3 items: prev/current/next) instead. The 5-item window is computed but never reaches the DOM. The requirement says "default 5" but only 3 items are rendered.

**Impact:** The app works correctly with 3 DOM items — fewer items is arguably better for performance. The configurable `DOM_WINDOW_SIZE` constant is unused at the render layer.

### INT-02: Sort/filter position-preserve broken (FILT-01/03/04/05) — low-medium

`News.tsx` computes `startIndex` to preserve the user's position when sort/filter changes, but `SnapFeed` has no `startIndex` prop and `useSnapFeed` always initializes `currentIndex` at 0. After any sort or filter change, the feed resets to the first card.

**Impact:** UX inconvenience — user loses their place when changing sort or filters. Not a functional blocker.

### INT-03: SortMode type duplication (PERF-02, FILT-01) — low

`useFeedState.ts` redeclares `SortMode` locally instead of importing from `@bts/shared`. Both definitions are identical today.

**Impact:** Silent type drift risk if the shared package is updated.
