## Directory Purposes

**src/**
- Purpose: All application code
- Contains: React components, pages, hooks, services, types, utilities, static data
- Key files: main.tsx (entry), App.tsx (routing root)

**src/components/**
- Purpose: Reusable, presentation-focused React components
- Contains: BiasFilter.tsx, EventCard.tsx, FeedCard.tsx, FeedFilter.tsx, MemberCard.tsx, Navbar.tsx, NewsCard.tsx, PhotoGallery.tsx, SwipeFeed.tsx
- Key files: Navbar.tsx (main navigation), FeedCard.tsx (feed item display), MemberCard.tsx (member grid card)

**src/pages/**
- Purpose: Full-page components tied to specific routes
- Contains: Home.tsx (landing), Members.tsx (members grid), MemberDetail.tsx (member profile), Tours.tsx (events), News.tsx (feed aggregator)
- Key files: News.tsx (most complex—orchestrates useFeed, filtering, view modes)

**src/hooks/**
- Purpose: Custom React hooks for shared stateful logic
- Contains: useBias.ts (member preference persistence), useFeed.ts (feed aggregation, caching, filtering)
- Key files: useFeed.ts (complex data fetching + caching + filtering logic)

**src/services/**
- Purpose: Business logic for fetching and transforming external data
- Contains: feeds.ts (primary—multi-source feed aggregation)
- Key files: feeds.ts (fetches Reddit, YouTube, news, Twitter; exports fetchAllFeedsIncremental for streaming results)

**src/types/**
- Purpose: TypeScript type definitions and constants
- Contains: feed.ts (FeedItem, FeedSource, BiasId types; MEMBER_KEYWORDS mapping)
- Key files: feed.ts (defines data contracts for entire feed system)

**src/data/**
- Purpose: Static data imported and rendered directly
- Contains: members.ts (7 members array with bios, photos, social), news.ts (fallback news items), events.ts (tour/event information)
- Key files: members.ts (core data for Members and MemberDetail pages)

**src/utils/**
- Purpose: Utility functions and helpers
- Contains: corsProxy.ts (CORS proxy fallback chain), xmlParser.ts (RSS/Atom parsing)
- Key files: corsProxy.ts (critical for working around browser CORS restrictions), xmlParser.ts (parses feed XML)

**src/assets/**
- Purpose: Static images and icons
- Contains: Compiled/copied image assets from build
- Key files: None—all images reference /public/members/*.jpg or /public/*.svg

**public/**
- Purpose: Static files served at root of deployed site
- Contains: manifest.json (PWA config), images (members/*.jpg, group.jpg), favicon.svg, apple-touch-icon.png
- Key files: manifest.json (PWA metadata), members/ directory (member photos)

**dist/**
- Purpose: Production build output
- Contains: Generated by `npm run build` command
- Key files: None—generated, not committed

## Key File Locations

**Configuration:**

**Core Logic:**
- `src/hooks/useFeed.ts`: Feed aggregation, caching, filtering logic (234 lines)
- `src/services/feeds.ts`: Multi-source fetch functions, keyword filtering (240 lines)
- `src/pages/News.tsx`: Page orchestration for feed UI, state management (127 lines)

**Testing:**
- No test files present in codebase

## Naming Conventions

**Files:**
- Components: PascalCase, .tsx extension (e.g., MemberCard.tsx, FeedFilter.tsx)
- Pages: PascalCase, .tsx extension (e.g., Home.tsx, News.tsx)
- Hooks: camelCase with "use" prefix, .ts extension (e.g., useBias.ts, useFeed.ts)
- Utils/Services: camelCase, .ts extension (e.g., corsProxy.ts, xmlParser.ts)
- Types: feed.ts (naming by domain)
- Data: Plural nouns, .ts extension (e.g., members.ts, events.ts, news.ts)

**Directories:**
- All lowercase, descriptive plurals: components/, pages/, hooks/, services/, types/, data/, utils/, assets/

**TypeScript Types:**
- Interfaces: PascalCase (e.g., FeedItem, Member, CacheData)
- Type aliases: PascalCase (e.g., FeedSource, BiasId)
- Constants: UPPER_SNAKE_CASE (e.g., STORAGE_KEY, CACHE_KEY, BTS_KEYWORDS)
- Functions: camelCase (e.g., fetchReddit, matchesBias, stripHtml)

**React Components:**
- Props interfaces: `{ComponentName}Props` (e.g., FeedFilterProps)
- State variables: camelCase (e.g., items, loading, error, biases)
- Event handlers: `on{Action}` (e.g., onChange, onClick, onToggle)

## Where to Add New Code

**New Feature:**
- Primary code: `src/pages/{FeatureName}.tsx` if a new page, or add to existing page
- Components: `src/components/{ComponentName}.tsx` if reusable across pages
- Hooks: `src/hooks/use{FeatureName}.ts` if extracting shared state logic
- Tests: `src/{location}.test.ts` (co-located with source file, though no tests currently exist)

**New Component/Module:**
- Implementation: `src/components/{PascalCaseName}.tsx`
- Export: Default export the component (e.g., `export default function MemberCard(...)`)
- Props: Define inline as interface `{ComponentName}Props` or as separate type

**Utilities:**
- Shared helpers: `src/utils/{camelCaseName}.ts`
- Domain-specific logic: `src/services/{camelCaseName}.ts`
- Types: Add to `src/types/feed.ts` (main type hub) or create new `src/types/{domain}.ts` if substantial

**Static Data:**
- New member info: Add to array in `src/data/members.ts`
- New static content: Create new file in `src/data/{plural}.ts` (e.g., `src/data/testimonials.ts`)
- Type definitions: Export types from `src/types/feed.ts` (already comprehensive)

## Special Directories

**node_modules/:**
- Purpose: npm dependencies
- Generated: Yes (run `npm install`)
- Committed: No

**dist/:**
- Purpose: Production build output
- Generated: Yes (run `npm run build`)
- Committed: No (.gitignore includes dist/)

**.git/:**
- Purpose: Version control metadata
- Generated: Yes (git init)
- Committed: N/A

**.planning/:**
- Purpose: Planning and analysis documents
- Generated: No (manually created)
- Committed: Yes

**public/:**
- Purpose: Static assets served at root URL during dev and production
- Generated: No (manual images)
- Committed: Yes (except derived images can be regenerated)
