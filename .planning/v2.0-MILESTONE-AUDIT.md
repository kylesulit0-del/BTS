## Cross-Phase Integration

### Server-Side Pipeline (Fully Wired)

All server-side wiring across Phases 5-7 is correct:
- 5 scrapers registered → `runAllScrapers()` → scheduler (20-min cron)
- `runAllScrapers()` → `runPipeline()` → `generateText` with structured output
- Pipeline → approved-only feed API with `contentType` filter
- `Status.tsx` → `/api/health/sources` and `/api/pipeline/stats`
- Shared config (`getBtsScrapingConfig`) consumed by all scrapers and fallback logic

**Frontend feed → Server API:** The News page uses client-side scrapers via `useFeed()` hook. It never calls `GET /api/feed`. This is by design — API-02 (dual-mode frontend) is Phase 8 work. As a result:
- LLM moderation output (approved items, content type labels) is server-side only
- Content type badges in FeedCard.tsx are present but `contentType` is always undefined from client-side scrapers
- Content type filter pills work but show 0 results for specific types
